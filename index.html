Metadata Schema Final Project

Chinese Name Recommendation Database for Foreigners

A documented XML metadata schema for describing how foreign personal names are transformed into culturally meaningful and pronounceable Chinese names.

1. Opening Narrative 2. Standards 3. Vocabulary 4. XSD 5. XML Samples

1. Opening Narrative

Chinese names assigned to non-Chinese speakers have historically been created through several strategies, including phonetic transcription, semantic translation, localization, and the preservation of names from other Han-character cultures. A usable name is therefore not only a string of characters. It also carries information about sound, meaning, cultural fit, identity, and the relationship between the original and newly created name.

This schema is intended for Chinese-language teachers, translators, international students, librarians, cultural mediators, name-recommendation services, and developers building searchable databases. It supports both human-readable documentation and machine validation.

The design evolved from a simple ten-element application profile into a more domain-specific XML schema. The final version adds original-language context, a complete Chinese-name display field, IPA pronunciation, and Hanzi radical information. It also separates general semantic borrowing from technical XSD implementation: concepts informed by Schema.org and FOAF are locally declared when no practical official XSD is available, while Dublin Core Terms is referenced as an external standard.

Theoretical consideration: a recommended Chinese name should be represented as a culturally situated identity label whose pronunciation, semantics, writing system, and naming method can all be documented independently.

2. Relevant Metadata Standards

Dublin Core Terms

Used directly for the record identifier and descriptive naming rationale because these are reusable cross-domain properties.

Schema.org

Informed the treatment of general name and nationality concepts in earlier drafts. The final XSD locally declares domain-specific versions for reliable validation.

FOAF

Informed the distinction between family name and given name. These concepts are implemented locally in the final XSD.

SKOS

Supported the design of definitions and controlled vocabularies, especially mapping type, difficulty, and cultural elements.

ISO 3166-1

Nationality values use two-letter uppercase country codes such as KR and US.

IPA

The International Phonetic Alphabet provides an optional pronunciation field beyond Pinyin.

4. XSD File

The XSD defines ChineseNameDatabase as the root element and allows one or more NameRecord elements. It includes documentation annotations, cardinality rules, controlled vocabularies, and design-intent comments.

Open XSD Directly Download XSD 
<?xml version="1.0" encoding="UTF-8"?>
<!-- 
     [Design Intent - Schema Architecture & OWL/RDF Handling]
     Design Intent: Following the professor's feedback from Draft 3, I needed to address how OWL/RDF-based vocabularies (like Schema.org and FOAF) are handled in an XSD environment. Since they do not provide official, accessible .xsd files, I adopted a "Local Redefinition Strategy": I borrowed their conceptual semantics but redefined them as local elements (e.g., <familyName>) under my targetNamespace to ensure validation works perfectly. 
     For standards that DO provide actual XSD files (like Dublin Core), I successfully used <xs:import> and `ref=` (e.g., dcterms:identifier, dcterms:description) to fulfill the rubric requirement. Additionally, I added domain-specific elements like `ipa` and `hanziRadical` to deepen the project's semantic richness based on peer and professor reviews.
     AI Usage: I prompted the AI to structure the XSD using xs:import for dcterms, and to translate my Draft Data Dictionary into locally validated elements with strict xs:enumeration restrictions. I specifically instructed the AI to add new domain-specific elements ('hanziRadical' and 'ipa') to reflect my design updates based on previous feedback. Finally, I manually reviewed the generated code to ensure all cardinality constraints (e.g., minOccurs="0" for familyName) perfectly matched my schema design.
-->
<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema"
           targetNamespace="http://www.skku.edu/metadata/cn/"
           xmlns="http://www.skku.edu/metadata/cn/"
           xmlns:dcterms="http://purl.org/dc/terms/"
           elementFormDefault="qualified">
    
    <!-- Design Intent: Importing an external schema with a valid schemaLocation to fulfill the requirement for referencing global external elements. -->
    <xs:import namespace="http://purl.org/dc/terms/" schemaLocation="https://www.dublincore.org/schemas/xmls/qdc/2008/02/11/dcterms.xsd"/>
    
    <xs:element name="ChineseNameDatabase">
        <xs:annotation>
            <xs:documentation>The root element containing multiple foreign Chinese name records.</xs:documentation>
        </xs:annotation>
        <xs:complexType>
            <xs:sequence>
                <xs:element name="NameRecord" maxOccurs="unbounded">
                    <xs:complexType>
                        <xs:sequence>
                            
                            <!-- 1. identifier -->
                            <!-- Design Intent: Using ref= to pull globally declared element from Dublin Core. minOccurs="1" makes it mandatory. -->
                            <xs:element ref="dcterms:identifier" minOccurs="1" maxOccurs="1">
                                <xs:annotation>
                                    <xs:documentation>데이터베이스 안에서 각각의 작명 데이터를 서로 구별하기 위해 시스템이 자동으로 부여하는 고유한 번호입니다.</xs:documentation>
                                </xs:annotation>
                            </xs:element>
                            
                            <!-- 2. originalName -->
                            <!-- Design Intent: Locally declared element borrowing concept from schema:name. -->
                            <xs:element name="originalName" type="xs:string" minOccurs="1" maxOccurs="1">
                                <xs:annotation>
                                    <xs:documentation>중국어 이름을 짓기 전, 외국인이 원래 사용하던 국가 기준의 본명 전체를 적습니다.</xs:documentation>
                                </xs:annotation>
                            </xs:element>

5. Sample XML Records

Each sample uses the same root structure and references cn_schema.xsd.

Sample 1: David → 戴伟

Localization Preserves part of the original sound while creating a natural Chinese surname-and-given-name combination.

Open XML
<?xml version="1.0" encoding="UTF-8"?>
<ChineseNameDatabase
    xmlns="http://www.skku.edu/metadata/cn/"
    xmlns:dcterms="http://purl.org/dc/terms/"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://www.skku.edu/metadata/cn/ cn_schema.xsd">
<NameRecord>
        <dcterms:identifier>CN-2026-0002</dcterms:identifier>
        <originalName>David</originalName>
        <inLanguage>English</inLanguage>
        <familyName>戴</familyName>
        <givenName>伟</givenName>
        <fullChineseName>戴伟</fullChineseName>
        <pinyin>Dài Wěi</pinyin>
        <ipa>[taɪ̯˥˧ weɪ̯˨˩˦]</ipa>
        <nationality>US</nationality>
        <mappingType>Localization</mappingType>
        <dcterms:description>원래 서양 이름인 'David'의 첫 발음 소리를 살리면서, 중국의 유명한 전통 성씨인 '대(戴)'씨로 연결했습니다. 위대하고 훌륭한 사람이 되라는 뜻입니다.</dcterms:description>
        <hanziRadical>亻</hanziRadical>
        <pronunciationDifficulty>Low</pronunciationDifficulty>
        <culturalElement>None</culturalElement>
    </NameRecord>
</ChineseNameDatabase>
